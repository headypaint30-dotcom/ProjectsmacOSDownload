Getting started

Install in 3 steps

One script sets up everything — folders, daemon, shell aliases. The only dependency is Homebrew, and fswatch installs automatically if missing.

1

Clone the repository

Download the project anywhere on your Mac.

git clone https://github.com/ProjectsmacOS-OFF/ProjectsmacOSDownload
cd ProjectsmacOSDownload
2

Run the installer

Creates ~/Projects/, copies scripts to ~/.projects-system/, registers the launchd daemon, and adds shell aliases to .zshrc.

bash install.sh
3

Reload your shell

Makes the new aliases available in your current terminal session.

source ~/.zshrc
Usage

Terminal commands

The daemon runs silently after login — you don't need to do anything. These commands let you control it and sort files manually.

projects

Navigate to ~/Projects

projects-sort

Sort all unsorted files now

projects-sort --dry-run

Preview without moving anything

projects-watch-status

Check if the daemon is running

projects-watch-start

Start the background daemon

projects-watch-stop

Stop the background daemon

Customize categories

Edit ~/.projects-system/config.sh — format is "CATEGORY|ext1,ext2". Rules run top-to-bottom, first match wins.

"Arduino|ino,pde"
"Config|env,yaml,toml,ini,cfg"

After editing, restart the daemon: projects-watch-stop && projects-watch-start

Logs

tail -f ~/Library/Logs/projects-classifier.log
Folder icons

Apply custom icons to Finder

The icons/ folder contains .icns files for each category. Applying them is optional but gives ~/Projects a polished look in Finder.

1

Open icon in Preview

Double-click Coding.icns — it opens in Preview automatically.

2

Copy the image

Press ⌘A then ⌘C to select and copy.

3

Open folder info

Right-click ~/Projects/Coding in Finder → Get Info ⌘I.

4

Paste the icon

Click the small folder icon in the top-left of the info panel (blue border appears), then press ⌘V. Repeat for each folder.

Install fileicon then apply all icons at once from the repo root:

brew install fileicon

fileicon set ~/Projects/_inbox  icons/_inbox.icns
fileicon set ~/Projects/Coding  icons/Coding.icns
fileicon set ~/Projects/Data    icons/Data.icns
fileicon set ~/Projects/Design  icons/Design.icns
fileicon set ~/Projects/Docs    icons/Docs.icns
fileicon set ~/Projects/Media   icons/Media.icns
fileicon set ~/Projects/Notes   icons/Notes.icns
fileicon set ~/Projects/Web     icons/Web.icns

To remove an icon: fileicon rm ~/Projects/Coding

Security

What runs on your machine

This tool is entirely local — no network requests, no telemetry, no remote servers. Here's what each part actually does.

watch.sh (daemon)

Runs as your user. Watches ~/Projects with fswatch. No ports, no network. Calls classify.sh on new files.

classify.sh

Moves files within ~/Projects only. Never touches files already in a subfolder. Adds a timestamp suffix on name conflicts.

config.sh

Sourced by every script. Treat it like any shell file in your home directory — don't paste untrusted content into it.

install.sh

Writes to ~/.projects-system/, .zshrc, and LaunchAgents/. Always inspect before running: cat install.sh

Found a vulnerability? Open a GitHub Security Advisory — please don't open a public issue for security bugs.

Compatibility

Supported configurations

Built and tested on Apple Silicon. Intel Macs work with a minor PATH fix documented below.

Apple Silicon M1 / M2 / M3 — macOS 12+
Fully tested · primary target
Intel Mac — macOS 12+
Replace /opt/homebrew/bin → /usr/local/bin in plist if daemon fails
macOS 11 Big Sur
Likely works · not tested
macOS 10.15 or earlier
Not supported — requires zsh as default shell
Linux
Not supported — launchd is macOS-only
Uninstall

Remove everything cleanly

Your ~/Projects/ folder and its contents are always left untouched.

launchctl unload ~/Library/LaunchAgents/com.user.projects-classifier.plist
rm ~/Library/LaunchAgents/com.user.projects-classifier.plist
rm -rf ~/.projects-system

Then open ~/.zshrc and delete the # Projects system block manually.
